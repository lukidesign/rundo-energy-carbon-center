## 一、项目概况

### 1.1 项目定位

RunDo Energy Carbon Center（原 Smart City Platform）面向建筑能耗管理场景，提供多层级视图（园区 → 建筑 → 楼层）的能耗可视化、AI 优化建议、设备监测及 ECM 系统集成功能。

### 1.2 技术架构

| 层级 | 技术选型 |
|------|---------|
| 框架 | React 18（UMD，Babel Standalone 浏览器端 JSX 编译） |
| 3D 渲染 | Three.js（WebGL，用于 Globe3D 地球仪与楼层模型） |
| 图表 | Chart.js 4.4.0 + chartjs-plugin-datalabels 2.2.0 |
| 部署形式 | **单文件 HTML**（zero-build，无需构建工具即可运行） |
| 设计同步 | Figma MCP 插件（html-to-design capture） |
| 版本管理 | Git → GitHub（lukidesign/rundo-energy-carbon-center） |
| 部署平台 | GitHub Pages（自动部署，push 后约 2–3 分钟生效） |

## 二、版本历史

### v4.0.0（2026-04-14）— 当前版本

**Globe3D 增强：**
- **TAR UMT 标记系统重构**：5 层标记架构（Glow halo + Core red dot + Invisible hit sphere + 2 Pulse Sprites + Gold label Sprite），Sprite 始终面向相机，金色渐变边框 + 红色 badge 的标签样式
- **Globe 自动旋转到 TAR UMT（A3）**：打开地球后 1.2 秒自动 fly-to 到马来西亚 TAR UMT 坐标（lat 3.2163°, lng 101.7291°），使用已有缓动系统（4%/帧 lerp + shortest-path wrapping）
- **Loading Screen**：WebGL 初始化期间显示 `city-overview.png` + 旋转 spinner，`ready` 状态由纹理加载完成驱动，双 CDN fallback（jsdelivr → unpkg）
- **Building 3D Drawer（左侧抽屉）**：400px 宽度，左侧滑入（cubic-bezier 0.35s），含楼层 tab 切换、`zbuilding3dimg/` 双层叠加图片、能耗数据卡片、图片加载失败降级处理
- **地球纹理预缓存**：`_earthTexCache` 全局变量，页面加载时即开始异步加载纹理，Globe3D mount 时直接复用
- **标签点击交互**：点击 TAR UMT 标签（`isLabel:true`）→ `onClose()` 返回主界面

**FloorView3D 数字孪生：**
- **全新 Three.js 楼层视图组件**：等距视角（52, 48, 52），5 个房间（Lecture Hall A、Computer Lab、Faculty Office、Seminar Room、Admin Office）+ 走廊
- **IoT 设备标记**：5 个设备（2 摄像头、2 温度传感器、1 报警器），Canvas 径向渐变 Sprite，状态着色（online=#00ff88, warning=#ffaa00, triggered=#ff3333），warning/triggered 脉冲动画
- **侧面板**：252px 宽，设备详情（点击选中）、楼层统计、SVG 迷你地图
- **性能优化**：DPR cap 1.5、共享 Material、`sortObjects=false`、60fps 帧率上限、`powerPreference:'high-performance'`
- **交互**：Raycaster 点击检测（5px 拖拽容差）、自定义 OrbitControls（极角限制、距离限制）

**全局优化：**
- **Error Boundary（D1）**：React 类组件包裹整个 App，`getDerivedStateFromError` + `componentDidCatch`，错误时显示友好 UI（⚠ 图标 + 错误信息 + 重新加载按钮），防止 JS 报错白屏
- **实时数据模拟增强（B4）**：波动幅度从 ±5% 提升到 ±6-15%，8% 概率触发 ±18% 突发尖峰，任意楼层均可触发告警（不再仅限 4F）
- **Chart.js 更新优化**：EnergyChart 改为 `chartRef` update-in-place 模式（不再 destroy/recreate），独立 unmount cleanup
- **FloorView3D marker 尺寸优化**：Sprite scale 7→3.5、canvas 48px、glow radius 28→20px、更紧凑的渐变 stops
