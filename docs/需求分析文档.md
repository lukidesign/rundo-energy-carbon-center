## 一、产品目标

### 1.1 核心价值

RunDo Energy Carbon Center（原 Smart City Platform）面向建筑能耗管理场景，通过多层级可视化视图（园区 → 建筑 → 楼层），帮助管理人员：

- **实时掌握**建筑各楼层能耗状态
- **直观对比**当前用量与 AI 优化后预期用量
- **快速响应**AI 建议，一键应用节能策略
- **追踪月度节省**效果（费用 / 能源 / 碳排放）
- **查看设备监测**数据（通过 VIEW 卡片联动 RunDo ECM 系统）

### 1.2 设计原则

| 原则 | 描述 |
|------|------|
| 信息密度适中 | 左侧面板承载数据，右侧视图承载空间感知 |
| 渐进式呈现 | 动画逐步引导用户注意力，避免信息轰炸 |
| 空间与数据联动 | 楼层视图点击 = 数据聚焦，保持上下文一致 |
| 深色沉浸风格 | 暗色背景 + 蓝绿荧光，符合智慧城市科技感 |
| 按需展示 | VIEW 设备卡片默认隐藏，由控制按钮显示 |

| 角色 | 描述 | 核心需求 |

### 3.4 Floor Tab — Three.js 3D 建筑模型

**功能描述：**
展示可交互的 Three.js 3D 建筑模型，支持拖拽旋转和缩放，楼层高亮与选择器联动。

| 需求项 | 描述 | 当前状态 |
|--------|------|---------|
| 3D 模型渲染 | WebGL 渲染，多楼层着色 | ✅ 已实现 |
| 拖拽旋转 | 鼠标拖拽旋转视角 | ✅ 已实现 |
| 滚轮缩放 | 鼠标滚轮控制远近 | ✅ 已实现 |
| 触控缩放 | 两指 pinch-zoom | ✅ 已实现 |
| 楼层高亮联动 | 选中楼层与 3D 模型高亮对应（使用 `f.id-1` 计算 Y 坐标） | ✅ 已实现 |
| AI 面板联动 | AI 打开时模型自动偏移居中 | ✅ 已实现 |
| 楼层选择器顺序 | 7F 在顶，1F 在底；选中高亮 pill 样式 | ✅ 已实现 |

**3D 楼层位置计算公式：**
```js
mesh.position.y = (f.id - 1) * (FH + FG) + FH / 2
```
> 注：`f.id` 为楼层号（1–7），不使用 FLOORS 数组索引，避免倒序导致高亮错位。
